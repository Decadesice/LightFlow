#### 一、整体界面布局

1. 页面整体为 **左右分栏布局** ：
   * 左侧为「对话列表 / 会话管理区」：宽度约占页面 1/5~1/4，底色为浅灰色系（低饱和度），顶部有产品 logo / 名称，下方展示历史会话列表（按时间倒序排列），每个会话项包含「会话标题（自动提取对话首句 / 用户自定义）+ 最近更新时间」，选中的会话项有高亮底色 / 边框标识；列表底部有「新建会话」按钮（图标 + 文字，如 “+ 新对话”）；支持会话项的右键操作（重命名、删除、收藏）。
   * 右侧为「核心对话区」：宽度占页面主体，底色为纯白色 / 极浅灰，分为上下两部分 —— 顶部是「对话内容展示区」，底部是「输入交互区」。
2. 响应式适配：
   * 桌面端：保持左右分栏；
   * 移动端 / 小屏设备：左侧会话栏可折叠为侧边抽屉（点击顶部 “菜单” 图标展开 / 收起），核心对话区占满屏幕。

#### 二、对话内容展示区交互与视觉

1. 消息气泡样式：
   * 用户消息：气泡底色为品牌主色（如蓝色 / 浅紫色），文字为白色，右对齐，气泡边角圆润，左侧可加用户头像（默认图标 / 自定义）；
   * AI 消息：气泡底色为浅灰色，文字为黑色 / 深灰色，左对齐，左侧加 AI 头像（品牌化图标），流式输出时文字逐字 / 逐句动态显示（模拟打字机效果），同时伴随轻微的 “光标闪烁” 动效；
   * 每条消息下方标注发送时间（小号灰色字体，hover 时显示完整时间，默认显示 “刚刚 / 1 分钟前” 等相对时间）。
2. 消息交互：
   * 单条消息 hover 时，右侧显示操作按钮（复制、点赞 / 踩、删除、引用回复）；
   * 支持选中消息文字进行复制；
   * AI 消息加载中：显示 “AI 正在思考…” 的提示文字 + 加载动效（如圆点跳动），流式输出时动效随文字生成逐步消失。
3. 对话区辅助功能：
   * 顶部有「清空对话」「导出对话」「会话设置」按钮；
   * 页面滚动：新消息生成时自动滚动到底部，用户手动上滑查看历史时暂停自动滚动，底部有 “回到底部” 按钮；
   * 历史消息：加载更多历史对话时，顶部显示 “加载中…” 的轻量提示。

#### 三、输入交互区设计

1. 输入框：
   * 多行文本输入框，高度可自适应（输入文字超过 1 行时自动增高，最大高度限制为屏幕 1/3）；
   * 输入框内有占位符文字：“输入你的问题，按回车发送…”；
   * 边框样式：未激活时浅灰色边框，激活时品牌主色边框，聚焦时有轻微阴影。
2. 功能按钮：
   * 输入框右侧：「发送」按钮（箭头图标 +“发送” 文字，未输入内容时置灰不可点击）；
   * 输入框左侧 / 下方：功能按钮组，包含「上传文件」（图标）、「表情 / 快捷指令」（图标）、「设置（如切换模型 / 参数）」（齿轮图标）；
   * 快捷键：支持「Ctrl/Command + Enter」发送消息，「Esc」清空输入框内容。
3. 输入辅助交互：
   * 输入过程中：底部显示 “支持 Markdown 格式” 的轻量提示（可关闭）；
   * 发送后：输入框自动清空，同时 AI 消息区开始加载流式内容；
   * 发送失败：输入框下方弹出红色提示 “发送失败，请重试”，并显示 “重试” 按钮。

#### 四、全局交互细节

1. 加载状态：
   * 首次进入 / 切换会话：页面中间显示轻量加载动效（如品牌 logo 旋转 / 圆点加载），加载完成后自动消失；
   * 所有按钮点击后：有轻微的 “按下” 反馈（如颜色变深 / 缩小 1%），避免重复点击。
2. 提示反馈：
   * 操作成功（如删除会话 / 复制成功）：页面右上角弹出浅色提示框（3 秒后自动消失）；
   * 错误提示：关键错误（如 API 调用失败）弹出居中提示框，包含 “重试”“关闭” 按钮。
3. 视觉风格：
   * 整体色调：以浅色系（白 / 浅灰）为底，品牌主色作为强调色，无过多鲜艳色彩，符合简洁、专业的 AI 产品调性；
   * 字体：正文使用无衬线字体（如思源黑体 / Inter），大小适中（14~16px），AI 和用户消息字体一致，提示文字字号更小（12px）；
   * 间距：元素间距均匀，气泡上下间距约 16px，左右内边距约 12px，避免拥挤。
